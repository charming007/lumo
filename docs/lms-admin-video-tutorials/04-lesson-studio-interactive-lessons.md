# Video 4 — Lesson Studio for interactive and option-based lessons

**Audience:** advanced lesson authors, curriculum editors, LMS admins  
**Target runtime:** 14-16 minutes  
**Goal:** teach the full lesson authoring flow for interactive, media-backed, and option-based lessons.

## Outcome

By the end of this video, the viewer should be able to:

- open the full Lesson Studio from the right entry point
- create a complete lesson pack, not just metadata
- build and sequence activity steps
- add choices, expected answers, evidence, media cues, and assessment items
- understand readiness blockers before publish

## Run of show

### Segment 1 — When to use Lesson Studio
**Narration:**
> Use Lesson Studio when the lesson needs the real payload: structured objectives, detailed steps, answer expectations, media cues, facilitation notes, and lesson-level assessment. If the lesson is interactive, option-based, or anything more than a bare record, this is the right tool.

### Segment 2 — Starting point
**Narration:**
> You can start from a template, duplicate an existing lesson, or build from scratch. Templates are usually the smart move. Starting blank is fine when you know exactly what you’re doing, but most people overestimate that.

### Segment 3 — Lesson shell
**Narration:**
> Start with the correct subject and module. Then set the title, duration, delivery mode, target age, and voice persona. These fields define the frame, but they do not make the lesson good on their own.

### Segment 4 — Objectives and learner preview
**Narration:**
> Add learning objectives that describe a real outcome. Then use the live preview panel to sanity-check how the lesson is taking shape from the learner point of view.

### Segment 5 — Localization and assessment pack
**Narration:**
> Add support language and localization notes that help the facilitator deliver the lesson well. Then build the lesson assessment pack with a title, kind, and concrete assessment items. Evidence should be explicit, not implied.

### Segment 6 — Activity spine
**Narration:**
> The activity spine is the main authoring surface. This is where you define the sequence the learner will actually experience. Add steps, move them, duplicate good ones, and remove weak ones.

### Segment 7 — Interactive steps
**Narration:**
> Each step should have a title, type, learner prompt, detail, timing, and evidence. For interactive lessons, the step type matters because it shapes how the learner responds.

### Segment 8 — Choices, expected answers, and media cues
**Narration:**
> For choice-based steps, add options using the line format shown in the form. Mark the correct option explicitly. Then define expected answers, evidence, facilitator notes, and any media cues needed for the step.

### Segment 9 — Readiness blockers
**Narration:**
> Before you approve or publish, check the blockers. Thin activity spine, weak objectives, missing assessment, bad timing, or a draft module upstream should stop the release. That friction is there for a reason.

### Segment 10 — Learner app flow
**Narration:**
> Once approved and assigned, the learner app consumes this authored payload. That means the quality of what you define here directly affects prompts, choices, response capture, completion evidence, and later progress tracking.

## Demo script for clicks

1. Open **Lesson Studio**.
2. Pick a **template** or explain duplicate mode.
3. Set **subject** and **module**.
4. Enter **lesson title**, **duration**, **delivery mode**, **target age**, and **voice persona**.
5. Add at least one **learning objective**.
6. Show the **live lesson preview** card.
7. Open **Localization** and explain support language plus notes.
8. Open **Assessment pack** and add at least two assessment items.
9. In **Activity spine**, add or edit a step.
10. Change the step type to **image choice** or **tap choice**.
11. Enter sample choices in this format:

```text
choice-1|Go to the market|correct|image|market.png
choice-2|Go to sleep|wrong|image|sleep.png
choice-3|Go to school|wrong|image|school.png
```

12. Add **Expected answers**.
13. Add **Evidence**.
14. Add **Facilitator notes**.
15. Add **Media cues** in this format:

```text
image|market.png
audio|prompt-1.mp3
```

16. Review **Inline readiness blockers**.
17. Explain why approval or publish should wait if blockers remain.

## Presenter notes

- Slow down here. This is the most operationally dense tutorial.
- Explain the difference between prompt, expected answer, evidence, and facilitator notes. That distinction matters.
- Say plainly that publishing a half-wired interactive lesson creates nonsense downstream.

---

## Companion narration block for the key interactive section

> For interactive steps, do not leave the teaching logic in your head. Put it in the lesson pack. The prompt tells the learner what to do. Expected answers define what counts as success. Evidence tells the facilitator what to confirm. Notes explain how to scaffold or retry. And if the step depends on images or audio, add the media cues here so the runtime is not guessing later.
