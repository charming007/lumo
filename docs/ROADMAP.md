# Lumo MVP Roadmap

## Phase 0 — Product framing (Week 1)
- finalize PRD
- define lesson taxonomy
- define learner session flow
- define shared-tablet operational model
- confirm visual direction from provided mockups

## Phase 1 — Core scaffolding (Weeks 2–3)
- scaffold Flutter learner app
- scaffold Next.js LMS
- scaffold NestJS API
- establish monorepo/workspace structure
- define database schema
- implement auth and roles

## Phase 2 — Core LMS + learner flows (Weeks 4–6)
- learner profile switching
- subject home screen
- lesson player shell
- student roster management
- attendance flow
- lesson assignments
- basic reporting

## Phase 3 — Offline learning loop (Weeks 7–8)
- local lesson pack storage
- local progress persistence
- sync queue implementation
- sync endpoints
- teacher dashboard progress views

## Phase 4 — AI-assisted workflow (Weeks 9–10)
- recommendation rules engine
- AI lesson draft suggestion flow
- review/approval screens
- analytics pipeline for lesson performance

## Phase 5 — Pilot hardening (Weeks 11–12)
- QA and device optimization
- content loading and audio testing
- monitoring and error handling
- pilot deployment prep
- facilitator training materials

## Pilot-ready definition
- learners can complete lessons offline
- progress sync works reliably online
- teachers can assign lessons and track progress
- reports can be exported for pilot stakeholders
- seed curriculum is available across three subjects
