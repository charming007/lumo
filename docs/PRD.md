# Lumo MVP PRD

## 1. Product summary

Lumo is a voice-powered AI learning platform for children who cannot reliably access school, literacy support, or internet connectivity. The MVP is designed for Almajiri children in northern Nigeria aged 8–12 using shared Android tablets in facilitated learning environments.

## 2. MVP goal

Deliver a pilot-ready platform that can run guided, voice-led foundational learning sessions offline and synchronize progress plus adaptive recommendations when connectivity is available.

## 3. Primary users

### Primary learner
- Almajiri children in northern Nigeria
- Age 8–12
- Low or no literacy
- Interact mainly through audio and simple touch

### Secondary users
- Teachers / facilitators
- Program coordinators
- NGO / donor / government stakeholders

## 4. Core problems

- Many children cannot read, so text-heavy edtech fails immediately
- Many children are not in formal school systems
- Connectivity is unreliable
- Existing tools are not culturally or operationally suited to informal learning environments
- Stakeholders need measurable outcomes, not just engagement

## 5. MVP scope

### Learner app
- Shared tablet learner profile switching
- Voice-first lesson navigation
- Subject selection
- Guided lesson session flow
- Audio playback and microphone interactions
- Basic progress/reward feedback
- Local offline storage
- Sync queue for events and progress

### LMS / admin
- Teacher login
- Student roster management
- Attendance tracking
- Lesson pack assignment
- Observation notes
- Progress dashboard
- Downloadable reports
- Approval flow for AI-generated lesson pack suggestions

### Content system
- Subject, unit, lesson, activity model
- UNESCO/UNICEF-aligned seed curriculum structure
- Human-reviewed content publishing
- Voice persona support

## 6. Subjects in MVP

1. Foundational English oral literacy
2. Basic numeracy
3. Hygiene, safety, and life skills

## 7. UX principles

- Zero reading dependence for the child experience
- Large touch targets
- Clear visual/audio cues
- Short sessions (5–10 minutes)
- Encouragement and reward loops
- Replayable prompts
- Minimal cognitive overload

## 8. Lesson session flow

1. Greeting / identify child
2. Subject / assigned lesson selection
3. Guided lesson begins
4. Listen-and-repeat or listen-and-answer interaction
5. Immediate corrective feedback
6. Micro-assessment
7. Completion summary
8. Progress saved locally
9. Sync when online

## 9. AI usage in MVP

### Included
- Difficulty adaptation using progress data
- Teacher-facing lesson draft suggestions
- Evaluation pipeline for lesson effectiveness
- Analytics-based recommendations

### Excluded from MVP core
- Fully open-ended child chat
- Autonomous curriculum rewriting
- Unreviewed AI-generated lesson publishing

## 10. Success metrics

### Learning
- Lesson completion rate
- Skill mastery rate
- Improvement in oral literacy tasks
- Improvement in numeracy tasks

### Usage
- Weekly active learners
- Average lessons completed per learner
- Sync success rate
- Teacher assignment completion rate

### Operational
- Offline completion success
- Tablet session stability
- Report generation success
- Facilitator adoption

## 11. Risks

- Speech recognition accuracy in target accents/environments
- Device limitations on low-end tablets
- Curriculum quality and localization gaps
- Overly broad scope
- Data privacy concerns for minors

## 12. Non-negotiables

- Child-safe constrained interactions
- Human-reviewed content governance
- Role-based LMS permissions
- Opt-in voice storage only
- Auditability for content changes and progress data
