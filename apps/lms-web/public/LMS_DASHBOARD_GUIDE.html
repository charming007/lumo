5. Statuses & blockers

Readiness is not a vibe

Module status
  • draft — exists, not release-safe
  • review — nearing sign-off
  • published — live-ready
Lesson status
  • draft
  • review
  • approved
  • published
Assessment status
  • draft
  • active
  • retired
Release-safe content needs all of this
  • Enough approved/published lessons to match the module’s planned count
  • A linked assessment gate
  • A sane module status

7. End-to-end system flow

How Content Library, English Studio, assignments, learner runtime, and rewards work together

This is the operating loop admins actually need to understand: curriculum structure becomes authorable lessons, authorable lessons become assignments, assignments become learner activity, and learner activity feeds progress and rewards back into the LMS.

Content Library
  = official curriculum spine
  = subject → strand → module → lesson → assessment gate
English Studio
  = English-specific authoring and readiness layer
Assignments
  = delivery wrapper for approved content
Learner runtime
  = completion, mastery, progression, and intervention signals
Rewards & Progression
  = XP, badges, levels, and admin follow-up views
StageWhat happensWhy admins should care
Curriculum structureContent Library defines the real subject, strand, module, lesson, and assessment relationships.If the structure is sloppy, assignments, progress, and rewards all inherit that mess.
English authoringEnglish Studio reads the module and assessment context, adds objectives, vocabulary, activity sequence, readiness checks, and inline blockers, then writes the lesson back into the live content lane.English quality improves without breaking the canonical curriculum structure.
Assignment deliveryApproved lessons get mapped to cohorts, pods, mallams, and due windows.Assignments are the delivery wrapper around real curriculum objects — not a separate content universe.
Learner runtimeLearners complete lessons and generate attendance, completion, mastery, progression, and next-module signals.This is the evidence layer admins use for workboards, interventions, and progression decisions.
Rewards feedbackXP, badges, and levels reflect verified milestones and manual corrections when needed.Rewards motivate and make momentum visible, but they should never replace assessment or mastery logic.
Admin loopDashboard, Progress, Rewards, learner pages, and workboards surface the results back to operators.The LMS becomes a feedback loop instead of a pile of disconnected admin pages.
The simplest mental model
  1. Content Library decides the structure.
  2. English Studio improves English lesson quality.
  3. Assignments deliver approved content.
  4. Learner runtime produces progress evidence.
  5. Rewards turn milestones into visible feedback.
  6. Dashboard, Progress, and Rewards show admins what to fix next.
Important discipline
  • Assessment and mastery decide progression safety.
  • Rewards reinforce effort and milestones.
  • Manual XP or badge adjustments are for recovery and correction, not for faking readiness.
8. Lesson creation

How admins create lessons

Path A — Quick lesson creation via Content Library
  1. Open Content Library
  2. Click Create Lesson
  3. Choose subject and module
  4. Enter title, duration, and mode
  5. Submit the lesson record
  6. Open Lesson Studio if the lesson needs full interactive authoring
Path B — English-specific authoring via English Studio
  1. Open English Studio
  2. Click Author English lesson
  3. Select module, title, duration, and mode
  4. Review objective, vocabulary, and generated activity spine
  5. Check readiness controls and publish state
  6. Create the lesson, then open Lesson Studio if you need deeper control
Path C — Full interactive authoring in Lesson Studio

Use Lesson Studio when the lesson needs the real payload: activity steps, choice options, expected answers, hints, media, lesson-level assessment items, and inline approval/publish blockers. This is the correct route for interactive or option-based lessons.

Interactive / option-based lesson walkthrough

  1. Start in the right lane: open Lesson Studio from Content Library or jump in from English Studio.
  2. Create the lesson shell: set subject, module, title, duration, mode, age band, and voice persona.
  3. Build the activity spine: add, duplicate, reorder, or remove steps. Each step needs a title, type, prompt, detail, duration, and evidence field.
  4. Add choices for interactive steps: for image-choice or tap-choice tasks, add one option per line in the format id|label|correct/wrong|mediaKind|mediaValue.
  5. Define expected answers and hints: use Expected answers for what counts as correct, Evidence for what staff should confirm, and Facilitator notes for hints, retries, and coaching moves.
  6. Build the assessment pack: add an assessment title, kind, and assessment items using prompt|evidence lines.
  7. Check readiness before publish: confirm title, duration, objectives, assessment, and activity spine are complete, and that the module and gate context are actually real.

Choice format and learner flow

choice-1|Go to the market|correct|image|market.png
choice-2|Go to sleep|wrong|image|sleep.png
choice-3|Go to school|wrong|image|school.png
How this reaches the learner app
  1. The lesson is saved into the module in Content Library.
  2. Ops can assign it to cohorts, pods, and mallams.
  3. On the learner tablet, the facilitator selects the learner and opens the assigned lesson.
  4. The app plays prompts, shows options or media, accepts spoken or tapped responses, gives feedback, and saves completion for sync.
9. Recommended workflow

How to run this LMS without shipping nonsense

Curriculum ops

  1. Create or confirm the subject
  2. Create strands
  3. Create modules with intended lesson counts
  4. Add assessment gates
  5. Create lessons
  6. Move lessons through review and approval
  7. Check Release blockers
  8. Only then publish and assign

English content teams

  1. Confirm the English module exists
  2. Confirm the module has an assessment gate
  3. Use English Studio to author
  4. Check readiness score and blockers
  5. Approve/publish only when checks pass
  6. Verify the lesson appears correctly in Content Library
